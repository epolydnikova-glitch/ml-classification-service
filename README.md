#Сервис классификации достижений студентов

Готовый веб-сервис на Flask, предсказывающий вероятность получения студентом стипендии на основе его достижений в учебной и/или внеучебной деятельности.    
Используется предобученная модель машинного обучения, загружаемая из файла awarded_model_pipeline.pkl.   
Данные вводятся через веб-форму, результат отображается сразу после отправки. История предсказаний хранится в базе данных SQLite.  

#Как запустить

Для работы требуется Docker.    
Если Docker ещё не установлен, скачайте и установите [Docker Desktop](https://www.docker.com/products/docker-desktop/).

1. Клонируйте репозиторий:  
   bash    
   git clone https://github.com/epolydnikova-glitch/ml-classification-service.git  
   cd ml-classification-service  

3. Запустите сервис одной командой:  
   bash    
   docker compose up   

5. Откройте в браузере http://localhost:5000.  

Сервис поднимется автоматически – фронтенд, бекенд и база данных будут готовы к работе.  
Никаких дополнительных настроек не требуется.  

#API

Страницы веб-интерфейса:

GET / – главная страница с формой для ввода данных и получения предсказания.  
GET /prediction – страница предсказания. Можно передать параметр ?edit=<id> для редактирования существующей записи.  
GET /history – страница со списком всех сохранённых предсказаний. Для каждой записи доступна кнопка удаления.  

Обработка данных и REST-эндпоинты:

POST /predict – основной эндпоинт. Принимает данные формы. Если запрос выполняется через AJAX, возвращает JSON с полями awarded_prediction, awarded_probability и record_id. Иначе перенаправляет на главную страницу. Поддерживает обновление записи, если передан record_id.  
POST /delete/<int:record_id> – удаляет запись с указанным ID (используется кнопкой на странице истории). После удаления перенаправляет на /history.  
GET /api/records/<int:record_id> – возвращает JSON-представление записи.  
DELETE /api/records/<int:record_id> – удаляет запись и возвращает JSON с подтверждением.  

Формат данных для POST /predict:

Все поля передаются как обычные данные формы. Полный список и точные названия полей смотрите в исходном коде формы на странице предсказания (templates/prediction.html).

Пример JSON-ответа (при AJAX-запросе):

json  
{  
  "awarded_prediction": "Да",  
  "awarded_probability": 87.45,  
  "record_id": 42  
}

#Примечания

База данных (database.db) создаётся автоматически в папке instance/ при первом запуске. Файл базы не включён в репозиторий, но сохраняется в volume, поэтому данные не теряются между перезапусками контейнера.

Модель awarded_model_pipeline.pkl уже включена в репозиторий – дополнительных загрузок не требуется.

Все действия (предсказание, просмотр истории, удаление записей) доступны через веб-интерфейс без необходимости работать с API напрямую.
